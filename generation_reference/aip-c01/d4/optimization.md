# AIP-C01 D4 - 운영 효율 및 최적화

> 스냅샷: 2026-07 기준  
> 출처 기반: `docs/AIP-C01 시험범위.txt`  
> 비중: 12%

## 범위 요약

D4는 GenAI 애플리케이션의 비용, 성능, 리소스 효율, 관찰성을 다룬다. 토큰 최적화, 모델 선택, 캐싱, 배치/처리량, 스트리밍, 벡터 검색 최적화, CloudWatch/Bedrock 호출 로그/X-Ray 기반 모니터링이 핵심이다.

## 작업 4.1: 비용 최적화 및 리소스 효율성

- 토큰 추정과 추적, 컨텍스트 창 최적화, 응답 크기 제어, 프롬프트 압축, 컨텍스트 프루닝, 응답 제한으로 비용을 줄인다.
- 비용 대비 성능, 쿼리 복잡성 기반 계층화된 FM, 추론 비용과 응답 품질 균형, 가격 대비 성능 비율로 모델 선택을 최적화한다.
- 배치 처리, 용량 계획, 사용률 모니터링, 오토 스케일링, 프로비저닝 처리량 최적화로 처리량을 높인다.
- 시맨틱 캐싱, 결과 핑거프린팅, 엣지 캐싱, 결정론적 요청 해싱, 프롬프트 캐싱으로 불필요한 FM 호출을 줄인다.

## 작업 4.2: 애플리케이션 성능 최적화

- 사전 계산, 지연 시간 최적화 Bedrock 모델, 병렬 요청, 스트리밍, 성능 벤치마킹으로 지연 시간과 사용자 경험을 개선한다.
- 인덱스 최적화, 쿼리 전처리, 하이브리드 검색, 사용자 지정 점수로 검색 관련성과 속도를 높인다.
- 토큰 처리 최적화, 배치 추론, 동시 모델 호출 관리로 처리량 병목을 해결한다.
- 모델별 파라미터, A/B 테스트, 온도, top-k, top-p를 요구 사항에 맞게 조정한다.
- 토큰 처리 용량 계획, 프롬프트/완료 패턴 사용률 모니터링, GenAI 트래픽 패턴 기반 오토 스케일링으로 리소스를 할당한다.
- API 호출 프로파일링, 벡터 DB 쿼리 최적화, LLM 추론 특화 지연 시간 감소, 효율적인 서비스 통신 패턴으로 워크플로 성능을 개선한다.

## 작업 4.3: GenAI 모니터링 시스템

- 운영 지표, 성능 추적, FM 상호 작용 추적, 사용자 지정 대시보드, 비즈니스 영향 지표로 전체 가시성을 확보한다.
- CloudWatch로 토큰 사용, 프롬프트 효과, 할루시네이션 비율, 응답 품질, 토큰 버스트, 응답 드리프트 이상 탐지를 수행한다.
- Bedrock Model Invocation Logs로 상세 요청/응답 분석, 성능 벤치마크, 비용 이상 탐지를 지원한다.
- 운영 지표 대시보드, 비즈니스 영향 시각화, 규정 준수 모니터링, 포렌식 추적성, 감사 로깅, 사용자 상호 작용 추적으로 통합 관찰성을 제공한다.
- 도구 호출 패턴, 성능 지표, 다중 에이전트 조정 추적, 사용 기준 이상 탐지로 도구 성능을 관리한다.
- 벡터 DB 성능 모니터링, 자동 인덱스 최적화, 데이터 품질 검증으로 벡터 저장소 운영 신뢰성을 높인다.
- 골든 데이터세트, 출력 비교, 추론 경로 추적, 특수 관찰성 파이프라인으로 GenAI 고유 실패 모드를 식별한다.

## 출제 포인트

- 비용 최적화 문제는 토큰, 캐싱, 모델 계층화, 프로비저닝 처리량, 배치 처리의 차이를 묻는다.
- 성능 문제는 스트리밍, 병렬화, 사전 계산, 모델 파라미터, 벡터 검색 최적화가 요구 조건과 맞는지 확인한다.
- 모니터링 문제는 CloudWatch, Bedrock 모델 호출 로깅, X-Ray, 대시보드, 비용 이상 탐지를 구분한다.
- 검색 성능과 비용은 벡터 저장소 선택, 검색 빈도, 관리형/서버리스 여부, 인프라 운영 부담을 함께 고려한다.
