# AIP-C01 D5 - 테스트, 검증, 문제 해결

## 범위 요약

D5는 GenAI 출력 평가, 모델 비교, 사용자 피드백, 품질 게이트, RAG/검색 평가, 에이전트 평가, 배포 검증, 문제 해결을 다룬다. 비중은 가장 작지만 실무적인 진단 문제가 많이 나올 수 있다.

## 작업 5.1: GenAI용 평가 시스템 구현

- 관련성, 사실적 정확성, 일관성, 유창성 같은 지표로 FM 출력 품질을 평가한다.
- Bedrock Model Evaluations, A/B 테스트, 카나리 테스트, 다중 모델 평가, 토큰 효율성, 지연 시간 대비 품질, 비용 성능 분석으로 모델 구성을 비교한다.
- 피드백 인터페이스, 등급 시스템, 응답 품질 주석 워크플로로 사용자 중심 평가를 구현한다.
- 지속 평가 워크플로, 회귀 테스트, 자동 품질 게이트로 성능 표준을 유지한다.
- RAG 평가, 평가형 LLM, 사람 피드백 수집으로 다양한 관점의 평가를 수행한다.
- 관련성 점수 매칭, 컨텍스트 매칭 검증, 검색 지연 시간 측정으로 정보 검색 구성 요소를 테스트한다.
- 작업 완료율, 도구 사용 효율성, Bedrock Agent 평가, 다단계 워크플로 추론 품질로 에이전트 성능을 평가한다.
- 시각화 도구, 자동 보고, 모델 비교 시각화로 이해 관계자에게 성과를 전달한다.
- 합성 사용자 워크플로, 할루시네이션 비율, 시맨틱 드리프트, 응답 일관성 검증으로 FM 업데이트 배포를 검증한다.

## 작업 5.2: GenAI 애플리케이션 문제 해결

- 컨텍스트 창 오버플로, 동적 청킹, 프롬프트 설계, 잘림 오류 분석으로 콘텐츠 처리 문제를 해결한다.
- 오류 로깅, 요청 검증, 응답 분석으로 GenAI 서비스 API 통합 문제를 진단한다.
- 프롬프트 테스트 프레임워크, 버전 비교, 체계적 개선으로 프롬프트 품질과 일관성 문제를 해결한다.
- 모델 응답 관련성 분석, 임베딩 품질 진단, 드리프트 모니터링, 벡터화 문제 해결, 청킹/전처리 개선, 벡터 검색 성능 최적화로 검색 문제를 해결한다.
- 템플릿 테스트, CloudWatch Logs, X-Ray, 스키마 검증, 체계적인 프롬프트 개선 워크플로로 프롬프트 유지 관리 문제를 해결한다.

## 진단형 문항 포인트

- “FM 호출은 성공하지만 관련 정보 없음”은 생성 실패가 아니라 검색, 임베딩, 청킹, 인덱스, 데이터 동기화 문제일 수 있다.
- 문서 인덱싱 임베딩 모델과 쿼리 임베딩 모델이 다르면 임베딩 드리프트로 검색 품질이 나빠질 수 있다.
- CloudWatch Logs에 오류가 없고 X-Ray가 성공을 보여도 품질 문제는 남을 수 있다.
- OpenSearch 클러스터 상태와 지연 시간이 정상이어도 검색 관련성, 임베딩 품질, 리랭킹, 청킹 문제는 별도로 봐야 한다.
- 모델 파라미터 문제와 검색 문제를 구분한다. 온도/top-k/top-p는 생성 다양성에 영향을 주지만 검색 컨텍스트 누락의 직접 원인이 아닐 수 있다.

## 출제 포인트

- 평가 문제는 자동 평가, 사람 평가, RAG 평가, 에이전트 평가, 배포 검증의 목적 차이를 묻는다.
- 문제 해결 문항은 관찰된 증상에서 원인을 좁히는 형식이 많다.
- 회귀 테스트, 품질 게이트, 카나리/A/B 테스트는 업데이트 안정성 요구와 연결한다.
- 컨텍스트 잘림, 청킹 오류, 임베딩 드리프트, API 권한 오류, 모델 파라미터 오류를 구분하는 문제가 적합하다.
